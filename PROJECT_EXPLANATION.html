Employee Management System (EMS)

A complete project overview and documentation

1. Project Overview

The Employee Management System is a professional full-stack web application. Its primary goal is to centralize and automate human resource tasks such as maintaining employee records, tracking attendance, and managing leave requests.

2. Technology Stack

Frontend

  • React.js: The framework used for the interactive user interface.
  • Ant Design: Used for high-quality components like tables and dashboards.
  • React Bootstrap: Used for responsive layouts and mobile compatibility.
  • Nivo / Chart.js: Used to create interactive pie charts and data visualizations.

Backend

  • Node.js & Express: The server environment and framework for the API.
  • JWT: Handles secure login and session management.
  • Bcrypt.js: Encrypts passwords for security.
  • Multer: Manages image uploads (Profile pictures).

Database

  • MySQL: A relational database for structured storage.
  • MySQL2: The connector that allows Node.js to talk to the database.

3. Key Features

🔐 Secure Authentication

The app uses a role-based system. Admins have full control over the organization, while Employees have access to their personal dashboard and leave history.

📊 Interactive Dashboard

The main screen shows real-time statistics: how many employees are in the system, how many are currently on leave, and a breakdown of different job positions.

📅 Leave Management

Employees can apply for leave by filling out a form with dates and reasons. Admins see a live list of "Pending" requests and can either Approve or Reject them with one click.

👤 Employee Directory

A searchable list of all staff members. Admins can add new users, edit existing ones, or remove them from the system.

4. How the App Works (The Logic)

Here is the step-by-step flow of the application:

  1. Login: User enters credentials. The Backend checks the MySQL database and returns a secure Token.
  2. Session: The Frontend saves this token and "remembers" if the user is an Admin or Employee.
  3. Requests: Every time the user clicks a button (like "Apply for Leave"), the Frontend sends the Token to the Backend to prove who they are.
  4. Data: The Backend updates the MySQL database and sends the new data back to the UI.

🚀 Quick Setup Guide

To run this project, follow these steps:

  • Config: Create a .env file in the backend folder with your DB_PASSWORD.
  • Admin: Run node create-admin.js to create your first login account.
  • Launch: Run npm run ems in the backend folder to start everything.
